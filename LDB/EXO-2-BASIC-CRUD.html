Exercice 2: Opérations CRUD

Ci-dessous sont présentés 20 exercices centrés sur les opérations CRUD (Create, Retrieve, Update, Delete) sur une seule table d'une base de données.
Ces exercices sont conçus pour vous aider à vous familiariser avec l'interface de phpMyAdmin ainsi qu'avec les opérations élémentaires des bases de données.
Ci-après vous trouverez des instructions pour chaque exercice, ainsi que des exemples de requêtes SQL pour vous aider à les résoudre.

Create Table Statement

CREATE TABLE `employee` ( `id` INT UNSIGNED AUTO_INCREMENT PRIMARY KEY, `firstname` VARCHAR(50) NOT NULL, `lastname` VARCHAR(50), `role` VARCHAR(100) NOT NULL, `department` VARCHAR(50) );

Rappels

Bonnes Pratiques des Opérations CRUD

  • Create: Lors de l'ajout de nouvelles données, assurez-vous que celles-ci respectent les contraintes de la table et n'introduisent pas de doublons ou d'incohérences.
  • Read: L'utilisation de requêtes optimisées et l'indexation appropriée des colonnes peuvent considérablement améliorer les performances de lecture.
  • Update: Lors de la modification des données, assurez-vous que les modifications respectent les contraintes de la table et ne créent pas de données incohérentes.
  • Delete: La suppression de données est une opération délicate, une technique appellée soft delete peut être utilisée pour marquer les données comme supprimées sans les effacer physiquement. Pour cela, une colonne deleted_at peut être ajoutée à la table, et les données marquées comme supprimées lorsqu'elles sont supprimées.
